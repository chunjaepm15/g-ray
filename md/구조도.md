# 🏗️ 리소스 구조화 및 협업 최적화 계획

팀원 4명이 서로의 코드를 건드리지 않고 **독립적으로 개발**할 수 있도록, HTML 내의 로직을 물리적 JS 파일로 분리하고 리소스를 체계화합니다.

## 📁 제안하는 프로젝트 구조

```text
g-ray-hrkang/
├── index.html            (로그인 관문 - 팀장/공통 관리)
├── main.html             (메인 페이지 UI)
├── teacher_problem.html  (문제 관리 UI)
├── teacher_student.html  (학생 결과 UI)
├── css/
│   └── style_v13.css     (🎨 공통 스타일 - 팀원 4 전용)
└── js/
    ├── curriculum_v13.js (📊 공통 데이터)
    ├── components/
    │   └── common.js     (🧱 공통 UI 컴포넌트 - 팀원 4 전용)
    └── pages/            (📂 페이지별 개별 로직 - 신규 생성)
        ├── main.js            (🧑‍🎓 팀원 1 전용: 메인 훈련 로직)
        ├── teacher_problem.js (🧑‍🏫 팀원 2 전용: 문제 설정 로직)

```

## 🛠️ 주요 변경 사항

### 1. [NEW] 개별 JS 파일 생성 (`js/pages/`)
- 각 HTML 파일에 들어있는 대규모 `<script type="text/babel">` 코드를 해당 JS 파일로 옮깁니다.
- **효과**: HTML 파일은 구조(HTML 태그)만 담당하게 되어, 로직 수정 시 Git 충돌이 발생하지 않습니다.

### 2. [MODIFY] HTML 파일 구조 경량화
- HTML 파일 상단에 필요한 JS 파일들만 로드하도록 간소화합니다.
```html
<!-- 예시 structure -->
<script src="js/curriculum_v13.js"></script>
<script src="js/components/common.js" type="text/babel"></script>
<script src="js/pages/main.js" type="text/babel"></script> <!-- 개별 로직 로드 -->
```

### 3. [NEW] 협업 가이드 (README) 생성
- 각 팀원이 담당해야 할 파일 경로와 주의 사항을 명시한 가이드 문서를 개발 환경에 비치합니다.
