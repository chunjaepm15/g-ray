# 프로젝트 진단 결과: 각 HTML 파일의 리소스 공유 현황 비교

안녕하세요! 팀원 여러분, 현재 진행 중인 3개의 주요 HTML 파일(`index.html`, `index-teacher.html`, `index-min.html`)에 대해서 각 파일이 어떤 자원(리소스 및 프로그램)을 사용하고 있는지 분석해 보았습니다.

우리가 개발 중인 프로그램의 구조를 더 쉽게 이해할 수 있도록, 세 파일이 공통으로 사용하는 리소스와 각 파일만이 별도로 사용하는 단독 리소스를 정리했습니다.

---

## 1. 세 파일이 모두 공유하고 있는 리소스 (공통 부분)
세 프로그램 모두 기본적인 디자인과 핵심 기능을 구성하기 위해 아래의 외부 라이브러리와 로컬 데이터들을 다 같이 공유하여 사용하고 있습니다.

* **외부 라이브러리 탑재 (CDN 방식)**
  * **폰트**: 구글 폰트 `Noto Sans KR` (전체적인 글꼴 디자인 통일)
  * **React 엔진**: `react.development.js`, `react-dom.development.js` (화면을 그리기 위한 핵심 프레임워크)
  * **Babel 컴파일러**: `babel.min.js` (리액트의 JSX 문법을 브라우저가 읽을 수 있도록 변환)
  * **Chart.js**: `chart.js` (통계나 차트를 그릴 때 사용)
* **내부 로컬 파일 (공통 연결)**
  * **디자인 파일 (CSS)**: `css/style_v13.css` (세 버전 모두 동일한 스타일과 디자인 규칙을 공유)
  * **데이터 파일 (JS)**: `js/curriculum_v13.js` (단원, 문장 등 학습과 관련된 공통 커리큘럼 데이터)

---

## 2. 각 파일이 공유하지 않고 단독으로 사용하는 별도 리소스 (개별 부분)
프로그램의 세부 앱 형태(일반 학생용 컴포넌트, 선생님용 컴포넌트, 혹은 통합형 컴포넌트)에 따라서 화면을 제어하는 핵심 로직(엔진) 코드는 서로 공유하지 않고 각자 다른 방식을 채택하고 있습니다.

* **📄 `index.html` (학생용/일반 메인 버전)**
  * **고유 연결 리소스**: `js/app_v13.js`
  * **설명**: 메인 애플리케이션 엔진 로직이 외부 분리 파일로 관리되며, 이를 불러와서 전체 앱을 동작시킵니다.

* **📄 `index-teacher.html` (교사용 관리자 버전)**
  * **고유 연결 리소스**: `js/teacher_admin_v13.js`
  * **설명**: 교사용 관리자 엔진을 별도로 두어, 선생님들에게 필요한 학급 분석, 단원 배포 등의 인터페이스만을 다루는 전용 로직 파일에 연결되어 있습니다.

* **📄 `index-min.html` (통합 단일 버전)**
  * **고유 리소스 (인라인 삽입)**: 외부에 .js 파일로 로직을 빼지 않고, HTML 내부의 `<script type="text/babel">` 태그 안에 **로직(App 컴포넌트 및 동작 코드) 전체가 직접 코딩**되어 있습니다.
  * **설명**: 파일 하나로 프로그램을 독립적으로 띄울 수 있도록 기획된 버전이라, 별도의 로컬 외부 스크립트 엔진 파일을 연결하지 않습니다.
