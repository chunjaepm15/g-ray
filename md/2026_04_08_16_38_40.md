# main.js 수정 내역 리포트

## 1. 텍스트 하이라이트 UI 자연스러운 문장형으로 개선
- **문제점:** 단어들이 개별 박스(div) 형태로 렌더링되어 간격이 떨어져 있고, 빈 공간까지 드래그 영역으로 분리 인식되어 마치 빈칸도 선택되는 것처럼 보이고 부자연스러움.
- **해결 방안:** 
  - 단어를 감싸는 컨테이너를 `div` 블럭에서 `span` 인라인 요소로 변경(`display: "inline-block"` 및 공백 조정)하여 자연스럽게 이어지는 하나의 문장 형태로 출력되게 함.
  - 단어 사이의 띄어쓰기 여백 영역에 `pointerEvents: "none"` 속성을 주어, 드래그 중인 마우스가 단어와 단어 사이의 가짜 빈칸 요소들을 각각의 독립 객체로 취급하지 않고 물 흐르듯 통과하도록 처리.

## 2. 클릭 후 드래그 문제(드래그 UX) 해결
- **문제점:** 단어 선택이 초기 클릭(`onClick`) 시 `dragRange`의 시작점으로 세팅되고 있어, "클릭"을 한 번 해야만 그 지점부터 드래그가 시작되는 불편함 존재.
- **해결 방안:** `onClick`을 `onMouseDown`으로 변경. 사용자가 마우스 클릭을 꾹 누르는 순간 즉시 범위를 설정하도록 하여, 다른 곳 클릭 없이 바로 누르면서 드래그가 매끄럽게 전개되도록 개선.

## 3. 배경 영역 클릭 시 하이라이트 선택 초기화 기능 추가
- **문제점:** 하이라이트를 실수로 칠했을 때, 이를 취소하기 위해 다시 원래 단어를 클릭해야 하거나 빈 곳을 클릭해도 사라지지 않는 문제.
- **해결 방안:** 문장 전체를 감싸는 부모 배경 컨테이너(`.sentence-view`)에 이벤트를 바인딩. `onMouseDown` 진행 중 사용자가 클릭한 대상이 단어가 아니라 순수 배경창 자신일 경우(`e.target === e.currentTarget`), 기존 선택 정보를 초기화(`start: null, end: null`)하도록 처리하여 편의성 증대.
