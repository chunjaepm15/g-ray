# G-ray 프로젝트 변경 내역 리포트

- **일시**: 2026년 4월 7일 14:53
- **병합 완료 대상**: `js/pages/main.js`, `student.html`

## 💡 병합의 목적 (Background)
외부에서 전달된 백업 파일 버전(로직 업데이트)과 현재 프로젝트의 원본 파일(디자인 및 UI 보존) 간의 충돌 문제를 해결했습니다. 
기존의 유려하게 구성되어 있던 페이지 디자인과 문제 풀이 가이드(난이도) 설정은 100% 보존하면서, 새롭게 고안된 핵심적인 "오답 처리 로직"만 선택적으로 이식하여 두 장점을 모두 취합했습니다.

## ✅ 주요 변경(병합) 사항 (Changes)

### 1. 2아웃(2회 오답) 정답 강제 노출 시스템 이식
- **투시 훈련(ACTIVITY)** 및 **유사 예문(SIMILAR)** 페이지 내에서 오답 횟수가 2회가 넘어갔을 경우 더 이상 진행이 막히지 않도록 정답을 시각적으로 공개하는 로직을 추가했습니다.
- 사용자가 구문 드래그를 2번 틀리면 **원래 정답의 드래그 범위가 자동으로 선택**되어 잡히게끔 보정되었습니다.
- 단어 배열, 접속사 선택, 영작 등의 예문을 2번째 틀렸을 때는, 사용자의 정답 박스에 **강제로 정답 텍스트와 배열이 자동 기입**되도록 처리했습니다.

### 2. 단계별 오답 가이드 기능 강화
- 1회 오답 시에는 단순 오답 표시 혹은 기존의 [💡 힌트 보기]를 활성화하도록 유도합니다.
- 2회 오답 아웃일 경우 단순하게 다음 문제로 넘어가지 않도록, "❌ 아쉽네요! 정답을 공개합니다." 라는 알림과 함께 **"💡 오답 가이드"** 를 명시적으로 하단에 뿌려주는 UI를 적용했습니다.

### 3. 백업 파일 내 불필요한 데이터 제거 및 원상 복구
- 병합 과정에서 난이도를 의도적으로 상향하기 위해 사라졌던 "한국어 해석 힌트"와 "가이드 텍스트(플레이스홀더)"를 원본처럼 다시 살려내어 학생들의 진입 장벽을 낮추었습니다.
- 파일 상단의 더미(Dummy) `CHUNK_DATA` 데이터 오염을 예방하기 위해, 백업 파일 데이터 대신 원본의 학습 데이터 포맷을 유지했습니다.
