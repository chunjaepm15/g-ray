# GrammarViewerPro 변경 이력 (Git Merge 지원용)

## [2026-04-06] - UI 개선 및 구문 분석 데이터 정교화

### 📝 요약
흰색 화면 출력 오류를 해결하고, 학습 분석 화면의 교육적 UI 요구사항 및 구문 분석 데이터의 정확도를 향상시켰습니다.

### 🚀 주요 변경 사항

#### 1. 자산 버전 정렬 (흰색 화면 오류 해결)
- **목표**: `index.html`이 참조하는 v13 자산 파일들과 실제 파일명을 일치시킴.
- **조치 사항**:
    - `/css/style_v9.css` → `/css/style_v13.css`로 변경
    - `/js/curriculum_v9.js` → `/js/curriculum_v13.js`로 변경
    - 혼선을 방지하기 위해 중복된 `/js/index.html` 파일 삭제

#### 2. 커리큘럼 데이터 정교화 (js/curriculum_v13.js)
- **목표**: 모든 문장에 대해 정확한 S+V+O 구조를 제공하고 한글 라벨을 표준화함.
- **조치 사항**:
    - **"Total" 역할 분할**: `role: "Total"`로 묶여 있던 문장들을 주어(S), 동사(V), 목적어(O), 보어(C), 수식어(M) 단위로 세분화하여 분리.
    - **한글 라벨 표준화**: 모든 `kor` 필드를 "주어(S)", "동사(V)", "목적어(O)", "보어(C)", "수식어(M)" 형식으로 업데이트하여 컬러 박스 내 영문 약어와 한국어가 병기되도록 개선.

#### 3. ANALYSIS(분석) 화면 UI/UX 개선
- **목표**: 단원 선택 및 문장 분석 과정의 가독성과 정확성 향상.
- **조치 사항**:
    - **아코디언 동작**: [문장 구조 투시], [핵심 문법 개념 정리] 섹션을 **기본적으로 접힘(collapsed)** 상태로 설정.
    - **문법 형식 표시 (S/V/C)**: 
        - 기존의 하드코딩된 화살표 SVG와 박스 형태를 제거하고 **데이터(`syntax_chunks`) 기반 동적 렌더링**으로 교체.
    - **동적 단어 설명 (Voca)**:
        - 현재 분석 중인 문장에서 **주요 단어를 동적으로 추출**하여 표시하는 로직 구현.

#### 4. 전체 문장 리스트 생성 및 데이터 동기화
- **목표**: 학습 콘텐츠의 검토 및 관리를 용이하게 하고, 불필요한 데이터를 정리함.
- **조치 사항**:
    - **문장 리스트 생성 (`sentences_list.md`)**: `curriculum_v13.js`에서 모든 영어/한국어 문장 쌍을 추출하여 문법 단원별로 정리한 마크다운 리스트 파일 생성.
    - **데이터 최적화 및 동기화**:
        - [x] **데이터 보강 (v13.1)**: `sentences_example.md` 기준 10개 문법 유닛별 각 4문장(총 40문장) 신규 추출 및 반영.
        - [x] **구문 분석 고도화**: 모든 문장에 대해 S(주어), V(동사), O(목적어), C(보어), M(수식어) 단위의 척크 분절 및 한국어 라벨("주어(S)" 등) 적용.
        - [x] **Unit 03 최적화**: 의미상 주어 구문을 "to verb" 지점에서 명확히 분절하여 학습 직관성 개선.
        - 사용자가 `sentences_list.md`에서 제외한 문법 문장들을 `js/curriculum_v13.js`에서도 일괄 삭제하여 데이터를 정제함.
        - 파일 끝부분에 남아있던 미세한 `Total` 역할 구문들을 완벽히 분리(S/V/O/M)하여 데이터의 일관성 확보.
        - 각 단원별 `total_matches` 수치를 실제 문장 수에 맞게 업데이트.
